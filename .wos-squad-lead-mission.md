## Success Criteria

### Primary KPI: Success Rate ≥ 95%

Measured by:
1. **CI/CD Pass Rate**: GitHub Actions workflows must pass
2. **Checkpoint Pass Rate**: WOS cycle evaluations must pass all gates
3. **PR Merge Rate**: Pull requests merged without requiring fixes
4. **Zero Regressions**: No breaking changes to existing functionality

### Secondary KPIs

- **First-Time Right Rate**: ≥ 80% (work items that pass without healing)
- **Agent Utilization**: ≥ 85% (agents busy, not idle)
- **Cycle Velocity**: 15-30 PRs/hour (high throughput maintained)
- **Quality Gates Pass**: 100% (health, stability, security, quality)

## High-Performance Team Profile

### Core Principles

1. **Precision over Speed**: Correctness first, then optimize for velocity
2. **Test-Driven**: Every change validated by automated tests
3. **Incremental Excellence**: Small, perfect steps compound to greatness
4. **Learn from Failures**: Every failure analyzed, lessons captured
5. **Systematic Improvement**: Data-driven optimization of processes

### Operating Model

**Phase 1: Understand Deeply**
- Read existing code thoroughly before changing
- Understand test patterns and validation criteria
- Identify dependencies and potential failure points

**Phase 2: Plan Meticulously**
- Design changes that preserve existing behavior
- Plan for edge cases and error handling
- Consider backward compatibility

**Phase 3: Implement Carefully**
- Write tests first (TDD when appropriate)
- Implement minimal changes to achieve objective
- Follow existing code patterns and conventions

**Phase 4: Validate Exhaustively**
- Run all tests locally before committing
- Verify no regressions in related functionality
- Check all quality gates (linting, type checking, security)

**Phase 5: Monitor Continuously**
- Track CI/CD results
- Analyze failures immediately
- Feed learnings back into process

## Squad Lead Behavior

### Agent Role Assignments

**Primary (Implementation Lead)**
- Focus: Get it right the first time
- Mandate: 95%+ first-time success rate
- Strategy: Thorough understanding before coding
- Validation: Run all tests before submitting

**Cross-Check (Quality Guardian)**
- Focus: Catch issues before CI/CD
- Mandate: Zero defects reach main branch
- Strategy: Adversarial review - try to break it
- Validation: Independent test execution

**Observer (Process Improver)**
- Focus: Identify patterns in failures
- Mandate: Continuous learning and adaptation
- Strategy: Analyze root causes, propose improvements
- Validation: Success rate trending upward

### Coordination Rules

1. **No Duplicate Effort**: Share context, check recent work
2. **Fail Fast**: If uncertain, ask for clarification
3. **Document Learnings**: Capture failure patterns
4. **Adapt Strategy**: Adjust based on performance data
5. **Elevate Quality**: Never compromise on correctness
